## 🔄 В процессе разработки

**Статус**: Готов интерфейс, нужна реализация

Нужно реализовать:
- BM25 поиск (использовать Lucene или похожее)

- Reranking (использовать моделю)
- Гибридную фузию результатов

**Статус**: Готов интерфейс, нужна реализация

- Интеграция с Neo4j/Kuzu

- Трассировка причинности

### 3. Остальные агенты

## ⏳ Планируется

### 1. REST API слой
Требуется создать Controllers для:
- `/api/memory/record` - запись событий
- `/api/memory/query` - запрос памяти
- `/api/memory/artifacts` - управление артефактами
- `/api/tasks` - управление задачами
- `/api/metrics` - метрики системы

### 2. CLI инструменты
- `memory-cli` - инструменты для управления системой
- Команды для отладки, мониторинга, тестирования

### 3. MCP интеграция
- Model Context Protocol сервер
- Подключение к AI агентам через MCP

### 4. Тестирование
- Unit тесты для всех компонентов
- Интеграционные тесты
- Тесты производительности (benchmarks)
- Регрессионные тесты

### 5. Kubernetes развертывание
- Helm charts
- ConfigMaps и Secrets
- Service discovery
- Health checks и metrics

### 6. Мониторинг в Azure
- Application Insights интеграция
- Azure Monitor интеграция
- Логирование в Azure

## 📝 Соглашения кода

### Именование
- Агенты: `{Domain}Agent` или `Default{Domain}Agent`
- Интерфейсы: `{Domain}Agent` (с суффиксом Agent)
- Реализации: `{Domain}AgentImpl` или `Default{Domain}Agent`
- Пакеты: `com.openclaw.memory.{component}`

### Структурирование класса
1. Логирование (Logger)
2. Поля (состояние, зависимости)
3. Конструктор
4. Имплементация интерфейса
5. Приватные помощники
6. Вложенные классы

### Документирование
- JavaDoc для публичных методов
- Инлайн комментарии для сложной логики
- Логирование ключевых операций

## 🔐 Требования безопасности

- Event Store append-only гарантирует аудит-trail
- Провенанс отслеживает источник каждого данных
- Conflict Resolution обеспечивает целостность
- Все артефакты проходят QA гейты
